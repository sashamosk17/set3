## Постановка задачи

Провести сравнительный анализ двух реализаций алгоритма MERGE SORT:

- Стандартной рекурсивной с выделением дополнительной памяти.
- Гибридной, которая на массивах малого размера (≤ threshold) переключается на INSERTION SORT.

Тестовые данные:

- Размеры массивов: от 500 до 100000 с шагом 100.
- Диапазон значений: от 0 до 6000.
- Типы массивов: случайные, отсортированные по убыванию, почти отсортированные.

Параметр threshold: 5, 10, 15, 20, 30, 50.

Класс `ArrayGenerator` генерирует три базовых массива длины 100000:

- Случайный массив с элементами из диапазона [0, 6000].
- Массив, отсортированный по невозрастанию.
- Почти отсортированный массив: сначала сортировка по возрастанию, затем 1% случайных обменов элементов.

Для требуемого размера n берётся префикс длины n из соответствующего базового массива.

Класс `SortTester` измеряет время работы алгоритма сортировки с помощью std::chrono::high_resolution_clock. Для каждой конфигурации (тип массива, размер, алгоритм, threshold) выполняется 5 запусков с усреднением результатов.
 

## Результаты и выводы

Графики показывают, что:

- Стандартный MERGE SORT демонстрирует ожидаемый рост времени пропорционально n log n.
- Гибридный алгоритм с threshold = 15–30 устойчиво быстрее стандартного на всех типах массивов.
- Оптимальный порог для случайных и обратно отсортированных массивов: 15–30.
- Для почти отсортированных массивов оптимум может быть выше (30–50), так как INSERTION SORT эффективен на частично упорядоченных данных.
- При слишком малом threshold (5) накладные расходы на частые вызовы INSERTION SORT снижают выигрыш.
- При слишком большом threshold (50) эффект гибридизации почти пропадает, время приближается к стандартному MERGE SORT.

Локальные всплески времени на графиках видимо связаны с внешними факторами.
